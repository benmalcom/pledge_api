/plegapi/v1/users

GET

Sending a get request to this API endpoint returns an array of JSON objects of all the users available on that resource, with each array element(i.e a user) in JSON format or it returns an empty array if no user is available on the resource.

Example
GET http://www.example.com/plegapi/v1/users.
RETURNS array of users [{user1},{user2},{user3}....] or an empty array [].

POST

Sending a post request to this API endpoint(mostly from a form field), is a request to create a new user. This request must be authorized, hence, the form field must contain parameters, which will also include an api_id and api_key field for authorization. If this request is successful, it returns the newly created user to the request sender, in JSON format.
REQUIRED PARAMETERS => first_name,last_name,gender,mobile,email also api_key and api_id(this is compulsory for authorization);
OPTIONAL PARAMETERS => lga_id,avatar

Example
POST http://www.example.com/plegapi/v1/users.
RETURNS a JSON object of the created user {"mobile_user_id":'new_user_id',"first_name":"*****","last_name":"****",.......}

/plegapi/v1/users/{id}

GET

Sending a get request to this API endpoint returns an object containing a user with the id supplied or an empty object if such user is non-existent.

Example
GET http://www.example.com/plegapi/v1/users/2
RETURNS an object in the form {"mobile_user_id":2,"first_name":"*****","last_name":"****"} or an empty object {}.

PUT

Sending a put request to this API endpoint(mostly from a form field), is a request to update an existing user. This request must be authorized, hence, the form field must contain parameters, which will also include an api_id and api_key field for authorization. If this request is successful, it returns the newly updated user data back to the request sender, in JSON format.
REQUIRED PARAMETERS => user parameters to update e.g first_name,last_name,gender,mobile,email,lga_id,avatar and also api_key and api_id(this is compulsory for authorization);
NOTE the id parameter in the url is the id of the user's data which you are updating.

Example
PUT http://www.example.com/plegapi/v1/users/3
RETURNS a JSON object of the updated user {"mobile_user_id":3,"first_name":"*****","last_name":"****",.......}
